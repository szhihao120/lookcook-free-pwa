# 版权与公开数据风险

## P0 数据来源

- 26 份菜谱：基于通用烹饪知识重新撰写；
- 菜谱数据许可：CC BY 4.0；
- 26 张封面：`scripts/build-recipe-data.mjs` 绘制的原创 SVG；
- 应用图标：项目原创 SVG，并栅格化为标准 PWA PNG；
- 代码：MIT。

没有复制旧项目、菜谱网站、社交平台、书籍或抓取数据中的文本和图片。

## 公开前检查

| 风险 | 当前状态 | 门禁 |
|---|---|---|
| 来源不明菜谱 | 已规避 | 每条必须有 source/author/license |
| 第三方图片 | 未使用 | 只允许自有、CC0 或明确授权素材 |
| 原作者项目代码归属 | 未直接复制 | 后续移植需保留原 MIT 声明 |
| 商标/人物/门店图片 | 未使用 | 人工审核 |
| API Key/密码 | 扫描通过 | `scan:secrets` + GitHub secret scanning |
| 医疗/营养宣称 | 仅定性标签 | 禁止无数据源的数值和疗效 |
| 菜谱事实相似 | 通用方法不可避免 | 文字保持独立表达 |

## 依赖许可

主要依赖均为可免费分发的开源软件：

- Vue、Vue Router、Vite、vite-plugin-pwa、Vitest、ESLint：MIT；
- idb：ISC；
- Workbox、Playwright、TypeScript：Apache-2.0。

完整机器清单保存在 `reports/evidence/dependency-licenses.json`。发布包保留根 LICENSE、
DATA_LICENSE 和依赖锁文件。

## 最高剩余风险

进入 P1 批量扩充时，人工或生成内容可能混入第三方原文/图片且来源字段被误填。必须
先做“来源许可必填 + 相似文本抽查 + 图片哈希/授权凭据 + 人工终审”，不能以数量目标
覆盖版权门禁。
