# 页面与交互说明

## 用户端页面

| 路由 | 页面 | P0 能力 |
|---|---|---|
| `/` | 首页 | 品牌首屏、搜索、快捷食材、推荐菜谱、隐私说明 |
| `/ingredients` | 食材选择 | 搜索、选择、清空、调味料过滤、已选标签 |
| `/recommendations` | 候选推荐 | 匹配排序、已有/缺少食材说明、返回重选 |
| `/recipes/:id` | 菜谱详情 | 人份换算、食材、步骤、营养标签、来源许可、收藏 |
| `/library` | 我的菜谱 | IndexedDB 收藏、最近浏览 |
| `/settings` | 设置与更新 | 安装说明、数据版本检查、离线与隐私状态 |

## 核心流程

1. 用户在首页直接搜索，或进入食材选择。
2. 选择番茄、鸡蛋等主要食材。
3. 推荐页按覆盖率、匹配数、缺失数和时间确定性排序。
4. 详情页默认显示 2 人份，可调整为 1–8 人份。
5. 收藏和浏览历史写入当前浏览器 IndexedDB。

## 空状态与错误

- 首次数据加载失败且无本地包：显示重试入口；
- 网络失败但已有本地包：显示离线提示并继续工作；
- 无搜索结果、无推荐、无收藏/历史都有独立空状态；
- 未知详情 ID 返回明确提示和首页入口。

## 本地管理端

`local-admin` 只在本机运行，支持：

- 载入当前完整菜谱包；
- 按菜名、ID、标签、食材搜索；
- 查看审核与许可状态；
- 编辑名称、来源、许可、版权状态和标签；
- 导入 JSON；
- 导出本地数据包。

保存按钮只改变当前浏览器会话，导出后才形成文件；没有后台 API 和自动上传。
