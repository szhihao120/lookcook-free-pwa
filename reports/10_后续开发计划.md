# 后续开发计划

## 阶段判断

建议进入 P1，但只在 GitHub Pages 发布成功、用户完成一次真实设备安装验收后开始。

## P1：数据与内容工具（建议 1–2 周）

- 将样例扩充到 100–200 道，仍采用小批量人工终审；
- 管理端增加 Schema 即时校验、重复检测、来源许可过滤；
- 维护食材同义词、季节、器具、过敏原标签；
- 生成增量数据包和变更日志；
- 增加数据回滚与旧版本兼容测试；
- 补 Android Chrome 和真实 iPhone Safari 设备验收。

退出条件：每道来源、许可、审核完整；0 个来源不明图片；搜索召回抽测通过。

## P2：体验增强

- 可选“小冰箱”本地库存；
- 购物清单；
- 饮食偏好和忌口，仅本机保存；
- Web Worker 搜索；
- 数据包按菜系/场景分包；
- 分享卡片使用原创封面。

## 延后或不做

- 在线 AI 接口与 API Key；
- 账号、评论、社区、投稿；
- 微信登录与云同步；
- 付费云服务、独立服务器和自定义域名；
- 纯离线 APK。

## 风险优先级

1. 版权与来源错误；
2. 批量内容质量和食品安全；
3. GitHub 免费额度与中国大陆网络波动；
4. iOS 对 PWA 安装、缓存回收和后台更新的限制；
5. IndexedDB 被用户清理导致收藏/历史丢失。

## P1 前置清单

- [x] GitHub Pages HTTPS 地址可访问；
- [ ] 用户在 Android Chrome 或 iPhone Safari 完成一次真实安装；
- [ ] 公开仓库 secret scanning 无告警；
- [ ] 确认菜谱数据署名显示方式；
- [ ] 选定 100 道 P1 菜谱范围与人工审核责任人。
